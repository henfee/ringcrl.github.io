Git使用指南，常用操作速查

发表于 2016-04-16   |  

安装Git

mac上安装完Xcode就会自动安装了Git

创建版本库

创建空目录

$mkdir name(创建文件夹)
$cd name(进入文件夹)
$pwd(显示当前目录)

将该目录变成Git仓库

$git init

将文件添加到版本库

1、将文件添加到仓库

$git add name.txt(文件)
$git add name/(文件夹)

2、将文件提交到仓库

$ git commit -m “描述性文字”

Git处理

查看Git情况

1、添加/提交状态，被修改过的文件的名字

$git status

2、查看修改的具体

$git diff README.md

版本回退

1、显示最近的3次提交，最新的版本在最上面

$git log

2、回退到某一个版本

$git reset - -hard 版本号

3、查看命令历史，找回某个版本的版本号

$git reflog

撤销修改

场景一：修改了文件，没有add

$git checkout - - README.md

场景二：修改了文件，并且add

1)回到场景一

$git reset HEAD README.md

2)按场景一操作

$git checkout - - README.md

场景三：已经提交

使用版本退回功能

删除/恢复文件

1、删除文件

$git rm README.md

2、恢复文件

$git checkout - - README.md

远程仓库

本地仓库与远程仓库链接

1、创建SSH Key

$ssh-keygen -t rss -C “youremail@email.com”
id_rsa(私密)
id_rsa.pub

2、在Github的Add SSH Key中的Key复制id_rsa.pub中的内容

添加到远程仓库

1、关联远程仓库

$ git remote add origin git@github.com:ringcrl/RingTalk.giat

2、推送本地到远程

第一次
$git push -u origin master
之后推送
$git push origin master

克隆远程仓库到本地

$ git clone git@github.com:ringcrl/RingTalk.git

分支管理

创建与合并分支

1、创建dev分支

$ git checkout -b dev
-b代表创建并切换
相当于
$git branch dev
$git checkout dev

2、查看当前分支

$ git branch

3、在当前分支dev进行提交

$git add README.md
$git commit -m “branch test”

4、切换回master分支

$git checkout master
会发现刚才添加的内容不见了
因为在dev分支上

5、把dev分支工作成果合并到master上

$git merge dev(dev是分支名，不是定值)

6、删除dev分支

$git branch -d dev

解决冲突

dev与master都有提交

1、查看Git情况 

添加/提交状态  
$git status
查看修改的具体
$git diff README.md

2、在vim中修改内容

3、进行提交

$git add README.md
$git commit -m “solve conflict”

禁用Fast forward模式

$ git merge --no-ff -m "merge with no-ff" dev
防止删除分支后丢掉分支信息

要离开当前工作现场

1、将当前工作环境储藏

$ git stash
此时用$git status查看工作区是干净的

2、如果要在master上修复bug，切换到master分支

$git checkout master

3、创建临时分支

$git checkout -b issue-101

4、提交修改

$git add README.md
$git commit -m “fix bug”

5、切换到master分支，完成合并，删除分支

$ git checkout master
$ git merge --no-ff -m "merged bug fix 101" issue-101
$ git branch -d issue-101

6、切换回原工作分支

$ git checkout dev

7、恢复工作现场 

查看工作现场
$ git stash list

如果有多个stash：
$ git stash apply stash@{0}
$ git stash pop stash@{0}

1、不删除stash内容的恢复
$ git stash apply
2、删除stash内容的恢复
$ git stash pop

销毁分支

一个已经提交了的分支，需要强制删除

$ git branch -D feature-vulcan

多人协作

1、查看远程库信息

$ git remote -v

2、推送分支

$ git push origin master
$ git push origin dev

3、「其他人」抓取分支（clone远程库默认只能看到master分支）

$ git branch
* master

4、创建远程origin的dev分支到本地

$ git checkout -b dev origin/dev

5、向origin/dev分支推送失败

$ git pull
如果还是失败：指定本地dev与远程origin/dev分支链接
$ git branch --set-upstream-to=origin/<branch> master
$ git pull

6、git pull成功，但是合并有冲突，则解决冲突： 

$git status

查看修改的具体

$git diff README.md

在vim中修改内容，进行提交

$git add README.md
$git commit -m “solve conflict”
